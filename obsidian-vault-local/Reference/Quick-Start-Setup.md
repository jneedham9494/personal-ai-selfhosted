## 🎯 Next Steps (5 Minutes to Test)

### Step 1: Install Dataview Plugin

**In Obsidian:**
1. Press `Cmd + ,` (Settings)
2. Click **Community Plugins** (left sidebar)
3. If you see a notice about safe mode, click **Turn on Community Plugins**
4. Click **Browse** button
5. Search for **"Dataview"**
6. Click **Install** on the Dataview plugin
7. Click **Enable**

### Step 2: Enable Dataview Settings

**Still in Settings:**
1. Scroll down left sidebar to find **Dataview** (under Community Plugins)
2. Enable these 3 checkboxes:
   - ✅ Enable JavaScript Queries
   - ✅ Enable Inline JavaScript Queries
   - ✅ Enable Inline Queries

### Step 3: Open Your Dashboard

1. Press `Cmd + O` (Quick Switcher)
2. Type: `Q4-Goals-Dashboard`
3. Press Enter

**What you should see:**
- ✅ Table showing 3 daily notes (Nov 7, 8, 9)
- ✅ Table showing weekly review (Week 45)
- ✅ List showing 1 workout day (Friday)
- ✅ Other queries (might be empty, that's OK)

**If tables are blank:**
- Check Step 2 again (Dataview settings enabled?)
- Try restarting Obsidian (Cmd + Q, then reopen)
- Check that Daily-Notes folder exists and has files

## 🛠️ Troubleshooting

### "I see code blocks, not tables"
**Problem:** Dataview plugin not installed or not enabled

**Fix:**
1. Settings → Community Plugins
2. Check if "Dataview" is in the list
3. If not, go back to Step 1
4. If yes, make sure toggle is **ON** (blue)

### "Tables are empty/blank"
**Problem:** Dataview settings not enabled OR files don't have frontmatter

**Fix Option 1 - Check Settings:**
1. Settings → Dataview
2. Verify all 3 checkboxes are enabled
3. Restart Obsidian

**Fix Option 2 - Check Files Exist:**
1. Open file explorer (left sidebar)
2. Look for `Daily-Notes/` folder
3. Should see 3 files: `2025-11-07-Thursday.md`, etc.
4. If missing, let me know - I'll recreate them

### "I see an error message"
**Problem:** Query syntax issue or folder path wrong

**Fix:**
1. Copy the exact error message
2. Tell me what it says
3. I'll help fix the query
