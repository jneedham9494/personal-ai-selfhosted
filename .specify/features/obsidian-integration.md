# Feature: Obsidian Vault Integration

## Status
Retroactive specification for existing feature

## Current Behavior
Service that manages the Obsidian vault structure, saves conversations with rich metadata, handles project/goal files, and provides search functionality. Also watches for file changes and processes prompt files.

## Implementation
- **Location:** [obsidian.js](src/services/obsidian.js)
- **Class:** `ObsidianService`
- **Lines:** 930

- `gray-matter` for YAML frontmatter parsing

- `ClaudeService` for AI-generated summaries/insights

## Dependents

## Acceptance Criteria (Current State)

### Initialization

- GIVEN valid vault path
  WHEN service initializes
  THEN vault existence verified, folder structure created, file watchers started

- GIVEN invalid vault path
  WHEN service initializes
  THEN error thrown with descriptive message

### Folder Structure

- GIVEN service initialization
  WHEN `createFolderStructure()` called
  THEN following folders created: AI-Conversations, Daily-Notes, Templates, Prompts, Projects

- GIVEN templates folder
  WHEN service initializes
  THEN AI-Conversation.md and Project.md templates created

### Conversation Saving

- GIVEN conversation data with messages
  WHEN `saveConversation()` called
  THEN markdown file created with frontmatter, formatted content, action items, and insights

- GIVEN conversation data
  WHEN saved
  THEN filename follows pattern: `YYYY-MM-DD-<topic>-HHMMSS.md`

- GIVEN ClaudeService available
  WHEN conversation saved
  THEN AI-generated summary and insights included

- GIVEN conversation saved
  WHEN daily note exists
  THEN conversation link appended to daily note

### Project Management

- GIVEN project name
  WHEN `createProject()` called with new name
  THEN project file created in Projects/ with template structure

- GIVEN project name that exists
  WHEN `createProject()` called
  THEN existing file path returned without modification

- GIVEN project name and progress
  WHEN `updateProjectProgress()` called
  THEN progress percentage updated, note added, status calculated

### Goal Management

- GIVEN goal name
  WHEN `createGoal()` called
  THEN goal file created with optional deadline, habit tracking section

- GIVEN goal with habit tracking enabled
  WHEN file created
  THEN weekly habit checkboxes included in file

### Search

- GIVEN search query
  WHEN `searchNotes()` called
  THEN all markdown files in AI-Conversations searched by content

- GIVEN search results
  WHEN returned
  THEN includes file path, frontmatter, and excerpt with query highlighted

### Daily Notes

- GIVEN today's date
  WHEN `getTodaysNote()` called and file exists
  THEN file content returned

- GIVEN today's date
  WHEN `getTodaysNote()` called and file missing
  THEN daily note created from template with goals, tasks, reflections sections

### File Watching

- GIVEN vault path
  WHEN watcher active
  THEN file add/change/delete events logged

- GIVEN Prompts/ folder
  WHEN new markdown file added
  THEN file processed by Claude, response saved as conversation, file archived

### Content Analysis

- GIVEN messages array
  WHEN `extractActionItems()` called
  THEN action items extracted from patterns like "I need to", "TODO:", "remind me to"

- GIVEN messages array
  WHEN `detectMood()` called
  THEN mood detected from keyword patterns (stressed, excited, focused, etc.)

- GIVEN topic and messages
  WHEN `categorizeConversation()` called
  THEN category assigned (work, personal, learning, health, finance, creative, general)

## Known Issues / Tech Debt

- [ ] **Huge file (930 lines)** - Should be split into modules
- [ ] **Simple keyword search** - No fuzzy matching or relevance scoring
- [ ] **Memory-intensive watcher** - Watches entire vault
- [ ] **No file locking** - Concurrent writes could corrupt files
- [ ] **Keyword-based analysis** - Should use Claude for better accuracy
- [ ] **No pagination** - Search returns all results
- [ ] **Template hardcoded** - Should be configurable
- [ ] **No error recovery** - Failed prompt processing moves to Error folder only
- [ ] **No vault backup** - Could lose data
